# Budget Friendly — Build Context & Handoff (2026-06-27)

Compacted context from the initial build session. For architecture/dev notes see
`CLAUDE.md`; strategy in `PRODUCT.md`; design tokens in `DESIGN.md`.

## What it is

A flexible **percentage budget calculator**. Start at 100%, carve it into any
number of named "buckets" (down to 1%); leftover lives in an explicit
**Unallocated** bucket. Enter an amount → live per-bucket figures + total. The
50/30/20-style rules are *starting templates*, not constraints. MVP is pure
client-side (localStorage); structured so accounts/bank features can be added
later without re-platforming.

Long-term vision (NOT built): connect a bank, auto-route incoming money to
accounts behind each label, sub-labels, lockable buckets, embeddable-in-bank view.

## Stack & layout

Next.js 16 (App Router) · React 19 · Tailwind v4 · Zustand (persist) · Motion.
Fully static build, no backend, no env required to run.

- `app/page.tsx` — **app shell**: slim top toolbar (brand + `ThemeSwitcher`, border-b) + full-height `<main>` workspace. `h-dvh`, no page scroll.
- `components/Calculator.tsx` — the workspace. **Desktop:** two flush panes split by a vertical divider — editor (`lg:flex-[1.4]`) + summary (`lg:flex-1`, `border-l`), each `overflow-y-auto`. **Mobile:** summary pane hidden; a fixed bottom bar (total + "Breakdown" button) opens a **bottom sheet** (`AnimatePresence` + `motion`) containing `<Summary onPick={closeSheet}/>`. Wraps everything in `<MotionConfig reducedMotion="user">`.
- `components/BucketBar.tsx` — the headline interaction (see below).
- `components/Summary.tsx` — flat fill component (no card): status pill, SVG donut, total, scrollable bucket list with select+delete rows, scroll shadows. Reused by desktop pane and mobile sheet. Accepts optional `onPick`.
- `components/BucketForm` (inside BucketBar) — the always-present editor form.
- `components/Select.tsx` — custom themed dropdown (replaces native `<select>` which can't be themed on dark). Used for currency + templates.
- `components/AmountInput.tsx`, `TemplatePicker.tsx`, `BucketTextAdd.tsx`, `ThemeSwitcher.tsx`.
- `lib/store.ts` (Zustand+persist, key `bf-store`), `lib/types.ts` (themes + Partition), `lib/format.ts` (currencies/format), `lib/templates.ts`, `lib/colors.ts`.

## Bucket bar interaction (core)

Buckets render as one tall horizontal bar; each is a colored segment with its
label inside (ink auto-picked from the color's OKLCH lightness, threshold 0.65).

- **Knobs** = thin full-height divider lines on each boundary seam (not floating pills). The bar container is `overflow-hidden` so the last knob sits flush (no overflow past the edge). The bar's border is drawn as a **top frame layer** so it sits over the knobs. Segment outer corners are rounded to match the bar so the selection outline follows the shape.
- **Drag/keyboard semantics:** the knob between two buckets **trades between just those two neighbours** (`adjustPair` in store — their combined % stays fixed, unallocated unchanged). The **last** knob (touching unallocated) grows/shrinks that bucket into the free space (`setPercent`, clamped — "block at available room").
- **Form** below the bar is **always rendered** (no collapse → no layout shift): placeholder text in row 1 when nothing selected; row 2 (color swatches, % input, delete) always present, disabled until a block is clicked. Contains **+ Add bucket**. New buckets prepend to the top and auto-focus their name.
- Selection (`selectedId`) lives in the **store** so the summary list (click row) and the bar stay in sync; summary rows also delete.

## Theming (14 themes, the signature feature)

Everything is a CSS custom-property token under `[data-theme="…"]` in
`app/globals.css`: color, type (family/weight/tracking), radius, border-width,
shadow style, motion (durations/easings), density. Components read tokens only.

Themes: candy, pastel, sunset, fintech, forest, retro, mono, **brutalist (default,
labeled "Neo Brutalism")**, midnight, noir, terminal, aurora, mocha, mono-inverse.
They're spread wide on every axis (radius 0→28, hard-offset/glow/flat/soft
shadows, 6 font families, bouncy/slow/snappy/smooth motion, varied density) so
switching has real impact.

- Default = **brutalist**. Set in: `lib/store.ts` initial `theme`, `app/layout.tsx` SSR `data-theme` + no-flash script default, and the `:root` block in `globals.css`.
- **CASCADE GOTCHA (caused a real bug):** the `:root` default block MUST be the **first** theme block in the file. `:root` has equal specificity to `[data-theme=...]`, so if it appears *after* some theme blocks, it overrides them (themes rendered as the default). Brutalist's `:root, [data-theme="brutalist"]` block is intentionally placed first, before all others.
- `ThemeSwitcher` = one pill: **name (tap → next)** │ **chevron (open dropdown)** │ **shuffle (random)**. Menu capped `max-h-[min(12.5rem,55vh)]` (~200px) and scrollable so it never covers the summary. `Select` menus use the same cap (uniform dropdown heights).
- `lastAddedId`/`selectedId` are in-memory (not persisted). `setTheme` writes `bf-theme` localStorage + `data-theme` + store, kept in sync.

## User preferences learned (treat as standing rules)

- **Just do exactly what's asked** — don't substitute your own interpretation, and don't half-measure repeated/explicit requests (e.g. dropdown height). The user got frustrated by both. Confirm significant/one-way-door decisions up front (stack, etc.) but then execute decisively.
- **No layout shift** is a hard constraint where stated (form must not collapse; verify dimensions before claiming done). Exception: the user later accepted minor theme-switch shift in exchange for maximal per-theme distinction.
- Deep, tasteful theming matters; "tokenize every design decision," not just color/font.
- Prefers app-like (Linear) feel over website feel.
- Uses design-taste skills: Emil Kowalski's `emil-design-eng` and the real **Impeccable** (`pbakaus/impeccable`, installed under `.agents/` — gitignored). Impeccable wrote PRODUCT.md/DESIGN.md.

## Dev gotchas

- **Stale dev CSS:** Turbopack frequently fails to recompile new `[data-theme]` blocks / global CSS edits — symptom: new themes render as the default. Fix: restart `npm run dev`, and if still stale, stop the server + `rm -rf .next` + restart. Always confirm new theme CSS in a fresh server (or check the production build, which is authoritative).
- Verify themes via the **production build** (`npm run build`) — it validates all CSS and renders the OG route. The minifier strips leading zeros (`0.5`→`.5`) so grep accordingly.
- Git warns LF→CRLF on Windows (harmless).
- `.agents/`, `.codex/`, `.impeccable/`, `skills-lock.json`, `_*.mjs` are gitignored (design tooling / scratch scripts). Playwright was added/removed ad hoc for screenshots — not a project dep.
